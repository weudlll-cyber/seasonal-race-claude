## OWASP Top 10 — Relevance Checklist for RaceArena

| # | Risk | Relevance | Mitigation |
|---|------|-----------|------------|
| A01 | Broken Access Control | **Medium** — race rooms must be isolated per session | Socket.IO rooms + JWT auth on all protected routes (server) |
| A02 | Cryptographic Failures | **Low** — no sensitive PII stored | localStorage data is non-sensitive config only; HTTPS in production |
| A03 | Injection | **Low** — no SQL in client; server uses parameterised `better-sqlite3` queries | Parameterised queries enforced; no raw SQL concatenation |
| A04 | Insecure Design | **Medium** — Game Master must not be able to cheat results | Race engine runs server-side; client only sends inputs |
| A05 | Security Misconfiguration | **Medium** — CORS, exposed dev endpoints | Dev server proxy must not be exposed in production; CORS restricted to known origins |
| A06 | Vulnerable Components | **Medium** — esbuild moderate vuln currently tracked | Tracked above; no high/critical findings |
| A07 | Auth Failures | **Medium** — JWT tokens control access | Short-lived JWTs; no persistent sessions; server validates on every request and socket event |
| A08 | Integrity Failures | **Low** — no CI/CD secrets in code | Secrets in GitHub Actions secrets store only; `.env` gitignored |
| A09 | Logging Failures | **Low** for now | `console.log` in client flagged as warning; server logging to be added |
| A10 | SSRF | **Low** — no outbound HTTP from client | Client calls own API only; no user-supplied URLs |
